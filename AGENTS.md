# AGENTS.md

## Cursor Cloud specific instructions

Sisidang is a thesis-administration monorepo: an Express + Prisma backend (`apps/backend`, port `5000`), a React + Vite + TypeScript frontend (`apps/frontend`, port `5173`), and PostgreSQL. The repo ships a Docker Compose setup (see `README.md`), but **Docker is not available in the Cloud VM**, so services are run natively against a local PostgreSQL 16 cluster and Node 22. Standard scripts live in each app's `package.json`.

### Services and how to run them

- **PostgreSQL**: does not auto-start on boot. Start it with `sudo pg_ctlcluster 16 main start` (check with `pg_lsclusters`). The `sisidang_user` role and `sisidang_db` database are already provisioned; connect via `postgresql://sisidang_user:sisidang_password@localhost:5432/sisidang_db`.
- **Backend**: `cd apps/backend && npm run dev` (nodemon on port 5000). Health: `GET http://localhost:5000/api/health-check`.
- **Frontend**: `cd apps/frontend && npm run dev` (Vite on port 5173, proxies API calls to `VITE_API_BASE_URL`).
- Seeded admin login: identifier `admin`, password `ChangeMe123!`.

### Non-obvious gotchas

- **Backend env var names**: `apps/backend/src/config/env.js` requires the *root* `.env.example` variable names (`BACKEND_PORT`, `MAX_FILE_SIZE_MB`, `CORS_ORIGIN`, `REFRESH_TOKEN_COOKIE_NAME`, `UPLOAD_DIR`, ...), **not** the names in `apps/backend/.env.example` (`PORT`, `MAX_UPLOAD_SIZE_MB`). The backend loads `.env` from its own directory (`apps/backend/.env`), so that file must contain the root-style names. This env file is gitignored and already set up in the VM.
- **DATABASE_URL host**: for native (non-Docker) runs the host must be `localhost`, not the Docker service name `postgres`. `apps/backend/.env` already uses `localhost`.
- **Migrations/seed** are already applied in the VM. To (re)apply after a DB reset: from `apps/backend` run `npx prisma migrate deploy` then `npm run db:seed` (both idempotent). Prisma client is regenerated by the startup update script.

### Lint / test / build

- Frontend lint: `npm run lint` (from `apps/frontend`) — the ESLint tooling works but the repo currently has pre-existing lint errors, so a non-zero exit is expected and not an environment problem.
- Frontend build: `npm run build` (runs `tsc -b` type-check + `vite build`) — passes.
- There is **no automated test suite** (no `test` scripts). `scripts/qa-smoke-test.ps1` is a PowerShell API smoke test; on Linux equivalent checks can be run with `curl` against the running backend.
- The backend has no lint step.
